# 「2027大选讨论」会话接入说明

> 把这一页整段粘贴到讨论会话里(或存为该会话的项目说明),那个会话以后就能一句话把讨论成果存进大选站。

---

## 给 Claude 的指令

你可以把本会话的讨论成果(研判分析、预测、预测结案、本周形势速览)存入 2027 大选站的数据库。方法:

在终端执行(heredoc 方式,JSON 按下面的格式拼):

```bash
cd /Users/junruzhang/Documents/GitHub/election-2027 && python3 scripts/analysis_to_db.py <<'JSON'
{ ...payload... }
JSON
```

### payload 格式(四个键都可选,有哪个传哪个)

```json
{
  "brief": "本周形势一段话——会覆盖网站置顶的紫色速览卡",
  "analyses": [
    {
      "title": "Attal 民调企稳的三个信号",
      "body_md": "正文,Markdown 格式,讨论中的分析整理成文",
      "author": "Gabby",
      "tags": ["民调", "Attal"],
      "evidence": [
        {"type": "url", "value": "https://example.com/poll", "note": "Ipsos 5月报告"}
      ]
    }
  ],
  "predictions": [
    {
      "statement": "Attal 会在 2026 年 9 月前正式宣布参选",
      "author": "Gabby",
      "confidence": 80,
      "deadline": "2026-09-01"
    }
  ],
  "resolutions": [
    {"id": 3, "status": "correct", "resolution_note": "6月10日官宣了"}
  ]
}
```

### 约定

- **预测要可验证**:一句话、有明确判断,最好带 deadline;`confidence` 是 0–100 的整数
- **结案 status** 只能是 `correct` / `wrong` / `partial`;结案前先问站主确认。预测的 `id` 在预测看板页每张卡片上有显示(#号),不确定就先查库:
  `python3 -c "..."` 或让站主报卡片编号
- **author** 写真名/昵称(谁说的算谁的战绩),用户没说就问,不要默认
- 脚本自带查重(研判按日期+标题,预测按内容+作者),重复执行安全
- 入库成功后提醒站主刷新网站即可看到

### 触发示例

用户说「把刚才这段分析存到站里」「这条预测记一下,我信心 85」「上周那条预测中了,结案」「更新一下本周速览」时,整理成上面的 JSON 并执行脚本。
